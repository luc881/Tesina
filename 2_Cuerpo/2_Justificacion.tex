\chapterstar{Justificación}

Actualmente Farmacia Selene lleva una gestión tradicional en la que cada artículo, venta y precio se registra de forma manual, lo que genera desorganización y pérdida de tiempo al realizar el corte mensual. Para atender esta problemática, se propone una solución tecnológica que unifica la gestión de inventario y ventas con el monitoreo ambiental, optimizando recursos y reduciendo tiempos operativos. A diferencia de los sistemas tradicionales, esta propuesta combina disciplinas como la computación, la electrónica, el IoT, las telecomunicaciones y el cómputo en la nube para ofrecer una plataforma integral, escalable y accesible desde cualquier lugar. Como se ilustra en la Fig.~\ref{fig:estructura_proyecto}, el sistema se organiza en tres capas interdependientes: la \textbf{capa frontend}, que constituye la interfaz desde la cual el personal registra ventas y consulta existencias; la \textbf{capa backend}, compuesta por el servidor, la API en FastAPI y la base de datos en PostgreSQL, que procesa cada solicitud y devuelve la respuesta correspondiente; y la \textbf{capa física}, que integra el microcontrolador ESP32 con el sensor AHT10 y el lector de códigos de barras para enviar mediciones periódicas de temperatura, humedad y productos al sistema. La integración de estos componentes genera alertas automatizadas con baja latencia, notificando al administrador ante cualquier variación que comprometa la calidad del inventario y registrando las acciones correctivas aplicadas para reducir el tiempo de reacción. La combinación de Python, FastAPI y PostgreSQL garantiza un desarrollo moderno y de alto rendimiento, asegurando que la plataforma evolucione con las necesidades futuras del negocio, mientras que la arquitectura en la nube asegura que los administradores accedan a la información actualizada desde cualquier lugar, con el fin de ofrecer a Farmacia Selene una herramienta confiable, adaptable y orientada a la optimización de sus operaciones.
